# 庫存與成本分析

mojo 幫你追蹤藥物庫存的使用狀況，並計算減重過程中的用藥成本。

## 藥物庫存

每次拿到新的藥物時，可以新增一筆庫存記錄：

- **藥物類型** — 注射筆或口服藥
- **總劑量 / 總數量** — 這盒藥的**總含量**（見下方說明）
- **購買金額** — 金額與幣別（支援 TWD、USD、JPY、EUR、CNY、KRW、HKD、SGD 等 16 種以上貨幣）
- **開封日期** — 何時開始使用

### 「總劑量」怎麼填？

| 藥物類型 | 填寫方式 | 範例 |
|---------|---------|------|
| 注射筆 | 整支筆的**總 mg 數** | 一支有 4 劑、每劑 5mg → 填 **20**（mg） |
| 口服藥 | 整盒的**總顆數** | 一盒 30 顆 → 填 **30**（顆） |

> ⚠️ 注意：注射筆的總劑量是整支筆的 mg 總量，**不是**每次的施打劑量。例如 Mounjaro 5mg 筆有 4 劑，總劑量應填 20mg，不是 5mg。如果填成每次劑量，庫存卡片上的進度條會很快就超過 100%，你可以到庫存管理修改。

庫存卡片會顯示進度條和已使用比例。用完的庫存可以封存。

### 在用藥記錄中選擇庫存

新增完庫存後，還需要在**用藥記錄**把每一次的施打或服藥**關聯到對應的庫存**，系統才會自動扣除用量並計算成本。

在新增或編輯用藥記錄的畫面上，會看到一個「庫存」區塊：

- 列出目前**這個藥物類型**所有未封存的庫存，點一下即可選取（換藥物類型時會自動重新載入）
- 如果還沒有可用的庫存，可以直接從這裡點「新增庫存」
- 下次新增用藥記錄時，會自動帶入你上次選的那筆庫存，不用每次重選
- 若當下不想關聯，可以選「不選擇庫存」；但這筆記錄**不會列入成本分析**，庫存用量也不會扣除

> ℹ️ 庫存選擇屬於 PRO 功能，非訂閱用戶會看到鎖定狀態。

## 成本分析

成本分析幫你了解減重過程中的用藥花費。這是訂閱用戶的進階功能。

**計算方式：**
1. 從你設定的「基準日」開始，只計算基準日之後的用藥記錄
2. 篩選出有關聯庫存且有填寫金額的記錄
3. 統計每支庫存的實際使用量，取「標示總劑量」與「實際使用量」的較大值作為有效總量
4. 根據每盒庫存的「金額 ÷ 有效總量」算出單位成本
5. 針對注射藥物：每次施打成本 = 劑量 × 單位成本
6. 針對口服藥物：每次服藥成本 = 一顆的成本
7. 將符合條件的用藥記錄成本加總，如遇多幣別則自動換算成你的顯示幣別

> 第 3 步的設計確保了即使總劑量填得比實際使用量低，總花費也不會超過庫存的購買金額。

**顯示內容：**
- 起始體重 → 目前體重 → 總共瘦了多少
- 累計用藥花費
- **每公斤成本** — 你瘦的每公斤花了多少錢

**多幣別支援：**
如果你的庫存使用不同幣別（例如在不同國家購買），mojo 會自動換算成你選擇的顯示幣別，使用即時匯率。

**常見疑問：**
- **金額跟實際花費對不上？** 請依序檢查：（1）庫存的購買金額欄位是空的（2）庫存的「總劑量」是否填錯（應填整支筆的總 mg，不是單次劑量）（3）部分用藥記錄沒有關聯到庫存。請到「藥物庫存」確認每筆都有填金額和正確的總劑量，並到「用藥記錄」逐筆確認都有關聯到對應的庫存。
- **基準日怎麼選？** 系統會根據你設定的基準日選項來決定起算點，只計算該日之後的用藥記錄。你可以在設定中調整。

**資料不足時：**
- 沒有基線體重 → 提示你需要先記錄體重
- 還沒瘦 → 顯示目前的累計花費，等有體重下降後才計算每公斤成本
- 用藥記錄沒有關聯庫存或沒填金額 → 該筆不計入成本
- 庫存購買金額填 0 → 視為免費試用，會正常計入（總花費為 0）
